## Tech Stack Used.

As usual, all our project at Zed Labs are built with state-of-the-art technologies. We're intentional about only giving
our clients the best for their money. It means a lot that they entrusted us with delivering their projects.

The core project build consists of two stand-alone front-end apps, and a backend.

### 1. The Front-end.

The project's frontends are being built using ReactJs and NextJs with - TailwindCSS for styling. This combination remains our
default choices for front-end development at Zed Labs. State management is currently being handled with Redux-ToolKit -
with Axios for data fetching. We hope to bring on SWR as well to enhance data fetching - especially for better data-revalidation
handling.

> Once and again, I'll honestly add that SWR seems to be quite underrated - just my humble opinion though.
>
> Don't know SWR? - [explore it here](https://swr.vercel.app)

Headless UI is our preferred UI Library. It's simple and perfect with just enough components to help us fast-track UI development in some areas.

### 2. The Backend.

The Backend is being handled with NodeJs/ExpressJs - my preferred stack for backend development(when stuffed with Typescript of course) prior
to meeting Python(Django - with the Django ninja framework) ðŸ¤“.

Node/Express fits perfectly well - since we got on the project with some initial progress. It only makes sense
to stick with the current stack to enable us deliver faster - with only necessary overhauls in needful areas/instances.

### 3. Cloud/Infra and Deployments.

For cloud service, AWS remains our go-to choice at Zed Labs. Everything so far is currently being handled on AWS -
from front-end deployments, to file storage, to EC2 VM(s), and more. However, the Database in this case is MongoDB.

> I plan to write an article shortly on why we prefer to use AWS as our cloud service provider at Zed Labs. Keep an eye on
> the Zed Labs blog just so you don't miss out when the post is released.

### 3. Payment Integration.

For payment integration, we hope to use Paystack. Just in case you don't know Paystack, Paystack is a stripe company, that has
been making massive strides in the African payment/fin-tech space.
